Detailed Description

The Entity class just describes some entity with a name. The entity can also have a parent entity. It may be null (there's no parent for the entity). Entity can also have children, but since this implements the Composite design pattern, Entity being the base class, Entity does not hold the children. The EntityComposite does. Entities can also be cloned using the clone() method, which is an implementation of the Prototype design pattern. Finally, Entities accept Visitors (
